## 3. Technical Standards: The "Fail-Close" Rule

All contributions and iterations of the HIK SDK must adhere to the following deterministic rules:

* **Zero Trust:** No data is accepted by the middleware unless its cryptographic "Trace" is verified against the ledger.
* **Fail-Close:** If a manifest is stripped, a signature is invalid, or the C2PA 2.3 metadata is missing, the system **must** terminate the process.
* **C2PA 2.3 Compliance:** All future releases must maintain backward compatibility with the C2PA 2.3 standard as the baseline for digital provenance.
* **Blockchain Anchoring:** Every critical assertion must be anchored to a Merkle Root on a supported ledger (Polygon/Ethereum) to prevent "offline" manipulation.
